# 펫플로우 테스트 MVP

제품의 장기 기준은 `docs/product-direction.md`를 따른다. v0.2는 관찰과 누적
건강 흐름까지 검증하며, v0.3부터 진료 전후의 반복 루프를 연결한다.

## 현재 v0.2 핵심 흐름

1. 이메일로 가입하고 연락용 휴대전화번호를 포함한 최소 테스터 정보에 동의한다.
2. 여러 반려동물을 등록하고 한 마리를 선택한다.
3. 오늘의 관찰 내용을 입력한다.
4. 오늘 기록의 안전 안내를 확인한다.
5. 여러 기록이 쌓인 건강 흐름과 병원 전달용 요약을 확인한다.

## 다음 v0.3 핵심 흐름

1. 관련 관찰을 하나의 사건으로 묶는다.
2. 병원에 전달할 짧은 요약을 공유한다.
3. 병원에서 받은 계획을 사용자가 직접 체크 항목으로 기록한다.
4. 같은 사건에 경과를 이어 쓰고 다음 상담에 다시 활용한다.

v0.3에서도 AI는 계획을 확정하거나 진단·처방을 만들지 않는다.

현재 구현됨:

- 기록 시 진행 중인 사건을 자동 생성하거나 재사용한다.
- 건강 흐름에서 사건별 기록 묶음과 진행 상태를 확인한다.
- 사건을 마무리하면 다음 기록부터 새 사건으로 시작한다.
- 사건의 구조화된 관찰을 한 장의 병원 전달 요약으로 확인하고 복사·공유한다.
- 병원에서 받은 계획을 사건별 최대 5개의 체크 항목으로 기록한다.
- 계획의 출처를 보호자 기록, 상태를 수의사 확인 전으로 고정해 표시한다.
- 계획 체크 상태를 저장하고 병원 전달 요약의 P 영역에 함께 넣는다.
- 같은 사건에 3일·7일·14일과 장기 30일·60일·90일 경과를 구조화해 기록한다.
- 경과를 사건 카드와 병원 전달 요약에 함께 표시한다.
- 메인 화면에서 최신 CHECK SCORE를 바로 확인한다.
- 참여코드를 입력한 인증 테스터만 GPT 기반 수의사 검토용 리포트를 생성한다.
- 같은 사건의 여러 날 기록, 계획, 3일·7일·14일과 장기 경과를 묶어 수의사 친화적인 검토용 보고서를 만든다.
- GPT 리포트는 수의사 확인 전 자료로 표시하며 진단·처방·약물·용량·치료 계획을 생성하지 않는다.
- GPT 리포트 사용량, 토큰, 유용성 점수, 지불의향을 파일럿 지표로 남긴다.

다음 구현 순서:

- 경과 기록 완료율과 재입력 이벤트 측정
- 수의사 유용성 평가와 누락 정보 피드백

## 포함

- 이메일 회원가입과 로그인
- 계정별 다중·다종 반려동물 관리
- 반려동물별 클라우드 기록 동기화
- 최근 14일 건강 흐름과 반복 변화 요약
- 건강 체크와 결정론적 위험 분류
- 오늘 기록과 누적 병원 전달용 요약, 복사, 모바일 공유
- 사건별 병원 계획 입력과 체크리스트
- 사건별 3일·7일·14일과 장기 30일·60일·90일 경과 기록
- 참여코드 기반 사건별 수의사 검토용 GPT 리포트 생성과 복사
- GPT 리포트 유용성·지불의향 피드백
- 익명 사용 통계와 테스터 피드백
- 테스터 관리용 Supabase 요약 뷰

## 제외

- 질병 진단, 처방, 약물·용량·치료 추천
- 주소, 정확한 위치, 실명 확인 정보, 반려동물 등록번호
- 결제, 푸시 알림, 동물병원 EMR 연동
- 의료서류 OCR과 이미지 진단
- 일반 사용자용 관리자 화면
- 병원 대시보드, RAG, 지식그래프

## 완료 기준

- 모바일에서 가입부터 리포트까지 완주할 수 있다.
- 로그인 사용자는 본인 반려동물만 조회·수정할 수 있다.
- 선택한 반려동물별로 기록이 분리되고 다른 기기에서도 다시 불러온다.
- 여러 기록에서 반복 증상과 위험 흐름이 결정론적으로 계산된다.
- API 키가 없어도 안전 규칙으로 리포트를 생성한다.
- lint, test, build, 원격 배포 검증이 통과한다.

## v0.3 완료 기준

- 한 사건 안에서 관찰, 공유 요약, 계획 체크, 경과 기록이 연결된다.
- 사용자 입력, AI 정리, 전문가 확인 상태가 섞이지 않는다.
- 모바일에서 새 경과를 1분 안에 이어 쓸 수 있다.
- 메인 화면에서 최신 CHECK SCORE를 바로 확인할 수 있다.
- 권한이 있는 테스터만 GPT 리포트를 생성하고 사용량이 관리된다.
