# 펫플로우 테스트 MVP

제품의 장기 기준은 `docs/product-direction.md`를 따른다. v0.2는 관찰과 누적
건강 흐름까지 검증하며, v0.3부터 진료 전후의 반복 루프를 연결한다.

## 현재 v0.2 핵심 흐름

1. 이메일로 가입하고 연락용 휴대전화번호를 포함한 최소 테스터 정보에 동의한다.
2. 여러 반려동물을 등록하고 한 마리를 선택한다.
3. 오늘의 관찰 내용을 입력한다.
4. 오늘 기록의 안전 안내를 확인한다.
5. 여러 기록이 쌓인 건강 흐름과 병원 전달용 요약을 확인한다.

## 다음 v0.3 핵심 흐름

1. 관련 관찰을 하나의 사건으로 묶는다.
2. 병원에 전달할 짧은 요약을 공유한다.
3. 병원에서 받은 계획을 사용자가 직접 체크 항목으로 기록한다.
4. 같은 사건에 경과를 이어 쓰고 다음 상담에 다시 활용한다.

v0.3에서도 AI는 계획을 확정하거나 진단·처방을 만들지 않는다.

## 포함

- 이메일 회원가입과 로그인
- 계정별 다중·다종 반려동물 관리
- 반려동물별 클라우드 기록 동기화
- 최근 14일 건강 흐름과 반복 변화 요약
- 건강 체크와 결정론적 위험 분류
- 오늘 기록과 누적 병원 전달용 요약, 복사, 모바일 공유
- 익명 사용 통계와 테스터 피드백
- 테스터 관리용 Supabase 요약 뷰

## 제외

- 질병 진단, 처방, 약물·용량·치료 추천
- 주소, 정확한 위치, 실명 확인 정보, 반려동물 등록번호
- 결제, 푸시 알림, 동물병원 EMR 연동
- 의료서류 OCR과 이미지 진단
- 일반 사용자용 관리자 화면
- 병원 대시보드, RAG, 지식그래프

## 완료 기준

- 모바일에서 가입부터 리포트까지 완주할 수 있다.
- 로그인 사용자는 본인 반려동물만 조회·수정할 수 있다.
- 선택한 반려동물별로 기록이 분리되고 다른 기기에서도 다시 불러온다.
- 여러 기록에서 반복 증상과 위험 흐름이 결정론적으로 계산된다.
- API 키가 없어도 안전 규칙으로 리포트를 생성한다.
- lint, test, build, 원격 배포 검증이 통과한다.

## v0.3 완료 기준

- 한 사건 안에서 관찰, 공유 요약, 계획 체크, 경과 기록이 연결된다.
- 사용자 입력, AI 정리, 전문가 확인 상태가 섞이지 않는다.
- 모바일에서 새 경과를 1분 안에 이어 쓸 수 있다.
