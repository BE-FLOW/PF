# 모바일 기능 이전 체크리스트

웹 MVP의 핵심 기능은 모바일 앱에서도 같은 데이터 계약으로 구현한다. 단, 모바일은
한 화면에 한 가지 결정을 두는 UX를 우선한다.

## 완료

- Expo 앱 골격과 앱 등록 설정
- Supabase Auth 로그인, 회원가입, 세션 유지
- 테스터 닉네임, 국내 휴대전화번호, 개인정보 동의 저장
- 반려동물 목록 조회
- 반려동물 등록, 수정, 선택
- 선택한 반려동물로 오늘 건강 기록 입력
- 결정론적 안전 분류와 결과 화면
- 서버 `/api/analyze` 저장, 실패 시 기기 내 기본 안전 분류
- 최근 기록 조회와 최근 14일 건강 흐름 요약
- 사진과 동영상 선택, Supabase Storage 업로드, 기록 첨부 메타데이터 연결
- 사건(`episode`)별 병원 공유 요약과 네이티브 공유
- 병원에서 받은 안내를 사건별 체크리스트로 저장하고 완료 상태 체크
- 같은 사건에 이어 쓰는 3일·7일·14일 경과 기록
- 경과 기록을 병원 공유 요약에 자동 반영
- 참여코드 기반 GPT 수의사 검토용 초안 생성과 네이티브 공유
- GPT 사용 후 유용성·지불의향 피드백 저장

- TestFlight와 Google Play 내부 테스트 배포

## 유지할 원칙

- AI가 진단, 처방, 약명, 용량, 치료 계획을 생성하지 않는다.
- OpenAI API 키와 service role key는 모바일 앱에 넣지 않는다.
- 모바일에는 공개 Supabase URL과 publishable key만 둔다.
- 건강 기록은 반려동물 아래 단건으로 흩뜨리지 않고 사건과 경과 흐름으로 이어간다.
