# PetFlow 제품 방향

이 문서는 `PetFlow와 BeFlow SOAP-LOOP 비즈니스 모델 보고서`를 바탕으로
현재 제품에 필요한 결정만 간결하게 고정한다.

## 한 문장 정의

PetFlow는 보호자의 관찰을 병원에 전달하기 좋은 형태로 정리하고, 병원에서
확정된 계획의 실행과 경과를 다시 기록해 다음 상담으로 연결하는 서비스다.

PetFlow는 AI 수의사나 진단 서비스가 아니다. BeFlow는 이 반복 구조를 다른
전문가 영역에서도 재사용하기 위한 내부 설계 원칙이며, 사용자 화면에는
PetFlow의 쉬운 언어만 사용한다.

## 핵심 루프

1. 보호자가 지금 관찰한 내용을 짧게 입력한다.
2. 안전 규칙으로 오늘 확인할 행동과 병원 문의 필요도를 안내한다.
3. 여러 관찰을 한 사건(`episode`)으로 묶어 병원 전달용 요약을 만든다.
4. 병원에서 받은 설명과 계획을 기록하되, 보호자 전달과 병원 직접 확인을 구분한다.
5. 기록된 계획을 작은 체크 항목으로 나누어 실행 여부를 기록한다.
6. 3일, 7일, 14일 경과를 같은 사건에 이어 기록한다.
7. 누적 경과를 다음 병원 상담 또는 다음 사건에 다시 사용한다.

장기 관찰이 필요한 사건은 30일, 60일, 90일 경과도 같은 사건에 이어 남긴다.
목표는 다른 병원이나 재진 때 보호자가 같은 설명을 반복하지 않게 하는 것이다.

사용자에게는 `SOAP`, `S/O/A/P` 같은 용어를 강요하지 않는다. 화면에서는
`관찰 기록`, `병원 전달 요약`, `병원에서 받은 계획`, `경과 기록`으로 표현한다.

## 데이터 원칙

- 기록은 반려동물 아래의 독립된 단건이 아니라 `subject -> episode -> input ->
  report -> expert review -> plan -> progress log` 흐름으로 연결한다.
- 모든 정보는 출처(`owner`, `ai`, `veterinarian`)와 확인 상태(`unreviewed`,
  `user_reported`, `expert_confirmed`)를 분리한다.
- AI가 만든 문장은 초안이며 전문가 확인 정보처럼 표시하지 않는다.
- 계획은 긴 메모로만 저장하지 않고, 실행 여부를 확인할 수 있는 작은 항목으로
  나눈다.
- PetFlow 화면과 용어는 반려동물에 맞추되 코어 식별자와 관계는 다른 도메인으로
  확장할 수 있게 설계한다.

## 개발 우선순위

현재 테스터 기준선:

- Google/Apple 계정 중심 로그인과 기존 이메일 계정 보조 경로
- 다중 반려동물, 클라우드 동기화, 연락용 국내 휴대전화번호 동의
- 보호자 관찰 입력, 사진·영상 첨부, 결정론적 안전 분류
- 오늘 기록, 최근 14일 건강 흐름, 기록 수정·삭제
- `episode` 기반 병원 전달 요약, 병원에서 받은 계획, 체크리스트
- 3일·7일·14일과 장기 30일·60일·90일 경과 기록
- 모든 로그인 사용자용 AI 수의사 검토 초안과 사용량·유용성 피드백 측정

다음 P0:

- 실제 보호자 테스트에서 기록 입력, 공유, 수정·삭제, 경과 재입력 흐름을 관찰한다.
- 앱 등록과 내부 테스트 배포 문구를 실제 스토어 상태에 맞춘다.
- 수의사 유용성 5점 평가와 누락 정보 피드백을 운영 지표로 확인한다.
- 보호자 5~20명과 병원 1~3곳의 짧은 파일럿을 준비한다.

지금 만들지 않는 것:

- 병원용 대형 대시보드와 EMR 양방향 연동
- RAG, 지식그래프, 자체 모델 학습, 범용 BeFlow 화면
- 결제, 보험, API 상품화, 복잡한 관리자 기능
- AI가 생성하는 진단, 처방, 약명, 용량, 치료 계획

AI 기반 리포트는 수의사가 사전 문진을 빠르게 검토하도록 돕는 초안이다. 모든 로그인
사용자에게 동일한 무료 월간 사용량을 제공하며 코드, 결제, 구독 또는 등급으로 기능을
잠그지 않는다. 리포트만으로 진료·진단을 완료하는 수준은 장기 목표이지만, 현재
파일럿에서는 질의응답 시간을 줄이는 검토 자료를 목표로 한다.

## UX 기준

- 한 화면에서는 한 가지 결정만 요청한다.
- 같은 정보는 다시 묻지 않고, 정상 상태는 한 번에 기록한다.
- 필수 입력은 다음 행동이나 안전 판단에 실제로 쓰이는 항목으로 제한한다.
- 빈 메뉴를 먼저 만들지 않고 반복 사용이 확인된 기능만 추가한다.
- 사용자가 완벽한 기록을 만들게 하지 않는다. 짧게 시작하고 나중에 이어 쓸 수
  있어야 한다.

## 안전과 개인정보

- PetFlow는 보호자 관찰과 병원에서 받은 계획을 정리하는 보조 도구다.
- 질병명 확정, 처방, 용량, 치료 권고는 생성하지 않는다.
- 초기 계획 입력은 사용자가 병원에서 실제로 받은 안내를 옮기는 방식으로만
  제공하며 `user_reported`로 저장한다. 병원이 직접 확인하지 않은 내용을
  `expert_confirmed`로 표시하지 않는다.
- 약명과 용량의 구조화는 별도 안전 결정 전까지 제외한다.
- 개인정보는 목적에 필요한 최소 범위만 수집하고, 사용자 데이터의 모델 학습은
  기본값을 사용하지 않음으로 둔다.
- 입력 원문, AI 정리, 전문가 확인을 UI와 데이터에서 명확히 구분한다.

## 파일럿 판단 지표

다음 수치는 확정 성과가 아니라 제품 적합성을 판단하기 위한 초기 가설이다.

| 지표 | 초기 기준 |
| --- | ---: |
| 관찰 입력 완료율 | 70% 이상 |
| 병원 전달 요약 공유율 | 60% 이상 |
| 수의사 유용성 점수 | 평균 4.0/5 이상 |
| 계획 체크 완료율 | 50% 이상 |
| 7일 내 경과 기록률 | 50% 이상 |
| 30일 내 재입력률 | 30% 이상 |

BeFlow 확장, 가격, 병원 과금 방식은 이 루프가 PetFlow에서 반복된 뒤 결정한다.
