# 펫플로우 UX 의사결정

## 제품 모토

사용자가 건강 정보를 완벽하게 정리하도록 요구하지 않는다. 지금 알고 있는 만큼만, 가능한 적은 입력으로, 다음 행동을 이해할 수 있게 한다.

## 2026-06-10 요청 검토

| 요청 | 결정 | 이유 |
| --- | --- | --- |
| 품종 | 채택 | 병원 전달 요약에 유용하다. 목록 제안과 직접 입력을 함께 제공한다. |
| 생년월일 | 채택 | 생애주기를 자동 계산할 수 있다. 선택 입력이며 네 자리 연도만 받는 날짜 컨트롤을 사용한다. |
| 성별·중성화 여부 | 채택 | 건강 맥락에 유용하지만 기록 시작을 막지 않도록 선택 입력으로 둔다. |
| 등록번호 | 제외 | 현재 핵심 흐름에 쓰이지 않고 개인정보와 입력 부담만 늘린다. |
| 타임라인·정리함 분리 | 제외 | 같은 기록을 다르게 부르는 혼란을 만든다. `지난 건강 기록`으로 통일한다. |
| 병원 메모 3분할 | 향후 단순화 | 도입 시 `병원에서 들은 설명`과 `메모·할 일` 두 칸만 사용한다. |
| 방문·체중·검사·예방접종 메뉴 | 보류 | 데이터가 없는 MVP에서 빈 메뉴와 수기 입력 부담을 만든다. 실제 반복 사용이 확인된 항목부터 추가한다. |
| 예방접종 세부 카테고리 | 보류 | 일정 알림과 데이터 신뢰성 설계가 먼저 필요하다. 단순 목록부터 검증한 뒤 확장한다. |

## UX 원칙

1. 같은 정보는 두 번 묻지 않는다.
2. 필수 항목은 핵심 흐름에 꼭 필요한 것만 둔다.
3. 정상 상태는 한 번의 선택으로 기록할 수 있어야 한다.
4. 전문 용어보다 보호자가 실제로 관찰한 표현을 사용한다.
5. 메뉴는 데이터와 사용 이유가 생긴 뒤 추가한다.
